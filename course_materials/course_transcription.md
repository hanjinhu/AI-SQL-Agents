### Chat completion models

Large language models are a class of foundation models, and they are designed to understand and generate text. The term LLM is often used more broadly to describe mixed models that can handle multiple types of inputs, such as text, images, video, and audio, and produce different types of outputs. In this course, we'll focus on the most common type, chat completion or response models. Those models take user input in text form, generate a response, and return it as text. Unlike conversation models, completion models typically do not have memory. This means each response is independent from the previous ones. The model input is often referred as a prompt. A prompt usually contains the user question or request, along with any additional context and instructions. In the next chapter, we'll dive into prompt engineering and explore how to optimize prompts efficiently. When model receive text input, it first breaks it down into smaller units called tokens, which are converted into numerical values, a process known as tokenization. Each model has its own limitation on the maximum number of tokens it can process. Most paid models charge based on the number of tokens being processed by the model, and you should be mindful of the associate cost of your model. In the context of AI agents, most models enable you to limit the maximum number of tokens that the model generated in the response. Finally, let's talk about reproducibility. Some LLMs allow you to control the level of randomness of the response through the temperature argument, which typically range between zero and two. Lower values make the model's response more deterministic, while higher values increase the randomness and creativity of the model. Throughout this course, we'll use the OpenAI API Python SDK. One of the great things about this SDK is that it has become an industry standout, and most LLMs are compatible with it. This means that if you would like to use an alternative model, either paid or free, you can still follow the course Codebase Workflow by making only minor adjustment to the API endpoint and model name. In the upcoming videos, we'll explore how to use different LLMs with the OpenAI API Python SDK. This includes paid options, like OpenAI GPT and Anthropic Load, as well as the Free Tier of Google Gemini. We will also look at how to run open-source models locally using Docker Model Runner.
